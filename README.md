CodeLayers Presents a comprehensive data analysis solution developed for the Envision Datathon, focusing on detecting and analyzing grey market activities in e-commerce platforms. The project leverages machine learning and statistical techniques to identify suspicious seller behavior and counterfeit product distribution patterns.

## ğŸ“Š Problem Statement

Grey market activities pose significant challenges to legitimate e-commerce platforms:
- Unauthorized sellers distributing genuine products outside official channels
- Counterfeit products masquerading as authentic items
- Price manipulation and unfair competition
- Customer trust erosion and brand reputation damage

## ğŸš€ Features

- **Anomaly Detection**: Identifies suspicious seller patterns and listing behaviors
- **Price Analysis**: Detects abnormal pricing strategies indicative of grey market activities
- **Seller Profiling**: Analyzes seller reputation, feedback patterns, and transaction history
- **Data Visualization**: Comprehensive visual insights into market trends and anomalies
- **Predictive Modeling**: Machine learning models to flag high-risk listings

## ğŸ’» Usage

### Round 1: Data Collection
1. Navigate to the script directory:
```bash
cd script
```

2. Run the data scraping script:
```bash
python main.py
```

### Round 2: Preprocessing & Visualization

1. Install additional dependencies (if not already installed):
```bash
pip install matplotlib seaborn
```

2. Run Round 2 preprocessing and visualization:
```bash
cd script
python run_round2.py
```

This will:
- Preprocess the raw data (`ecommerce_grey_market_data.csv`)
- Generate comprehensive visualizations
- Create a processed dataset (`processed_data.csv`)
- Save all visualizations to the `visualizations/` folder

3. Create Dashboard:
   - Follow the instructions in `DASHBOARD_GUIDE.md`
   - Use `processed_data.csv` as the data source
   - Create dashboard using Power BI or Tableau

## ğŸ“ˆ Dataset

### Raw Data (`ecommerce_grey_market_data.csv`)
The raw dataset contains e-commerce transaction data with features including:
- Seller information and ratings
- Product details and pricing
- Transaction timestamps
- Customer feedback metrics

### Processed Data (`processed_data.csv`) - Round 2
The processed dataset includes:
- Cleaned numeric columns (price, MRP, ratings)
- Derived features (price/MRP ratio, suspicious pricing flags)
- Feature engineering (rating quality score, review indicators)
- Ready for dashboard creation and analysis
